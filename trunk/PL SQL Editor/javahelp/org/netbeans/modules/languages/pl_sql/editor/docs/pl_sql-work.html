Working with PL/SQL code

To edit PL/SQL code of existing object you should locate it in Explorer and choose Edit from Popup Menu. First time the program will ask you where to save object code in the local file system. Next time when you would like to edit the same object program will automatically overwrite local file with the PL/SQL object code from the database. To change location of local file representing your database object you should choose Properties from the object Popup Menu and modify Local File property.
To create a new object you should locate appropriate object type in Explorer and choose Add from Popup Menu. Program asks for new object name and where to save object code in the local file system. Then program automatically compiles your new code in the database and shows it in the editor window.
Invalid database objects are marked with “x” icon, object name is in red color.
To compile PL/SQL object within the database use Compile from object Popup Menu. To compile PL/SQL object from your local file select Compile Local File from object Popup Menu. This action will save your PL/SQL code in the local file system and then compile it in the database. If this action is disabled you need to set the Local File property.
To drop an existing database object you should click on Delete from object Popup Menu. Database object will be dropped, but corresponding local file (if exists) will be only closed. So you have a chance to restore deleted object later.

You can adjust the fonts that are used in the PL/SQL Source Editor and the way colors and background highlighting are used to represent syntactic elements of your code.

  1. Choose Tools -> Options and click the Fonts & Colors panel.
  2. In the Languages drop-down list, select text/pl_sql.
  3. Select a syntax element in the Category list.
  4. Use the Font, Foreground, Background, Effects, and Effect Color fields to change the appearance of that code element.
